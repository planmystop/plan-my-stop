# Spacing System

> The Spacing System defines consistent spacing across the entire Plan My Stop platform. Proper spacing improves readability, establishes hierarchy, enhances usability, and creates a premium visual experience.

---

# Purpose

Spacing is invisible.

Users rarely notice good spacing,

but they immediately notice poor spacing.

The spacing system creates

- Rhythm
- Balance
- Consistency
- Readability
- Premium Quality

Professional

Whitespace is a feature.

Not empty space.

---

# Spacing Scale

Use a consistent spacing scale.

```
spacing.none

spacing.xs

spacing.sm

spacing.md

spacing.lg

spacing.xl

spacing.2xl

spacing.3xl

spacing.4xl

spacing.5xl
```

Never invent custom spacing values.

---

# Token Usage

Every margin and padding must use

```
spacing.*

```

Never use

```
margin:17px

padding:13px
```

Only design tokens.

---

# Component Spacing

Each component defines

- Internal Padding
- External Margin
- Gap Between Elements

Components should never manage page layout.

---

# Card Spacing

Every card contains

- Outer Margin
- Inner Padding
- Header Gap
- Content Gap
- Footer Gap

Cards should breathe.

---

# Form Spacing

Forms require consistent spacing between

- Labels
- Inputs
- Helper Text
- Validation Messages
- Buttons
- Sections

Users should never wonder

which field belongs to which label.

---

# Button Spacing

Buttons require

- Internal Padding
- Gap Between Icon and Text
- Gap Between Adjacent Buttons

Touch targets should remain comfortable.

---

# List Spacing

Lists define spacing between

- Items
- Sections
- Headers
- Dividers

Dense information should remain readable.

---

# Navigation Spacing

Apply consistent spacing to

- Top Navigation
- Side Navigation
- Bottom Navigation
- Navigation Items
- Icons
- Labels

Navigation should feel balanced.

---

# Grid Spacing

Spacing should separate

- Columns
- Rows
- Cards
- Panels
- Widgets

Grid rhythm creates visual harmony.

---

# Section Spacing

Every major section should have

- Top Margin
- Bottom Margin
- Internal Padding

Sections should feel clearly separated.

---

# Content Width

Limit reading width.

Long text should never stretch across large screens.

Readable layouts improve comprehension.

---

# Responsive Spacing

Spacing adapts for

Mobile

↓

Tablet

↓

Desktop

↓

Large Displays

Smaller screens require tighter spacing,

not cramped layouts.

---

# Touch Targets

Maintain adequate spacing between

- Buttons
- Icons
- Links
- Form Controls

Prevent accidental taps.

---

# AI Components

AI cards include spacing for

- Recommendation
- Explanation
- Confidence
- Actions

Information should be easy to scan.

---

# Journey Timeline

Timeline spacing separates

- Stops
- Events
- Notes
- AI Suggestions
- Travel Segments

Timeline readability is critical.

---

# Dashboard Layout

Dashboard panels maintain

- Uniform Gaps
- Consistent Padding
- Visual Rhythm

Avoid crowded dashboards.

---

# Empty States

Empty states include generous spacing around

- Illustration
- Heading
- Description
- Primary Action

Whitespace improves focus.

---

# Modal Dialogs

Dialogs require spacing between

- Title
- Description
- Content
- Actions

Content should never touch dialog edges.

---

# Tables

Tables maintain spacing for

- Headers
- Cells
- Rows
- Actions

Dense information should remain readable.

---

# Accessibility

Spacing supports

- Large Text
- Zoom
- Screen Readers
- Touch Accessibility

Layouts should remain usable at all supported scales.

---

# Performance

Spacing should be implemented using

- Design Tokens
- CSS Variables
- Component Tokens

Avoid duplicated spacing definitions.
