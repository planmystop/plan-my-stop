# Map Experience

> The map is not the product. The journey is. This document defines how maps should support intelligent journey planning rather than simple navigation.

---

# Purpose

Traditional map applications answer

"How do I get there?"

Plan My Stop answers

"What is the best way to experience this journey?"

The map exists to help users make better decisions.

---

# Vision

The map should become

The Journey Intelligence Canvas.

It should continuously explain

- What is happening
- What will happen
- What should happen next

---

# Design Philosophy

Maps should not overwhelm users.

Display only what is useful.

Hide complexity.

Reveal details progressively.

---

# Primary Goals

The map should help users

- Plan
- Explore
- Compare
- Decide
- Travel
- Discover

Navigation is only one capability.
