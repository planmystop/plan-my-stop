# Typography

> Typography is the voice of the interface. It establishes hierarchy, improves readability, communicates importance, and reinforces the premium identity of Plan My Stop. Every piece of text should have a clear purpose and a defined semantic role.

---

# Purpose

Typography exists to help users

Read

↓

Understand

↓

Decide

↓

Act

Typography should disappear into the experience.

Users should never struggle to read information.

---

# Vision

Typography should feel

Modern

↓

Elegant

↓

Professional

↓

Friendly

↓

Accessible

↓

Timeless

The interface should feel calm and effortless to read.

---

# Philosophy

Meaning

↓

Hierarchy

↓

Readability

↓

Consistency

↓

Accessibility

Typography should communicate before decoration.

---

# Core Principles

Typography should be

- Readable
- Accessible
- Consistent
- Responsive
- Scalable
- Semantic
- Premium
- International

---

# Font Strategy

Use

One Primary Font Family

↓

One Monospace Font

↓

Optional Display Font (Marketing Only)

Avoid multiple UI fonts.

Consistency builds trust.

---

# Primary Font

Used for

- Headings
- Body Text
- Navigation
- Buttons
- Forms
- Dialogs
- Tables

The primary font should support

- Latin
- Arabic
- Hindi
- Chinese
- Japanese
- Korean

Future global expansion should require no redesign.

---

# Monospace Font

Used only for

- Coordinates
- Debug Information
- Technical Data
- Logs
- Developer Screens

Never use monospace for normal UI.

---

# Typography Hierarchy

```
Display

↓

Heading

↓

Subheading

↓

Section Title

↓

Body

↓

Supporting Text

↓

Caption

↓

Overline
```

Hierarchy should be immediately obvious.

---

# Display Text

Used only for

- Landing Pages
- Marketing
- Hero Sections
- Empty States

Avoid Display styles inside application workflows.

---

# Heading Styles

Headings communicate structure.

Support

Heading 1

Heading 2

Heading 3

Heading 4

Heading 5

Heading 6

Each level should have a clearly defined purpose.

---

# Body Text

Body text should be

Easy to read

↓

Comfortable

↓

Consistent

Never sacrifice readability for visual style.

---

# Supporting Text

Used for

- Descriptions
- AI Explanations
- Helper Text
- Tooltips
- Instructions

Supporting text should never compete with primary content.

---

# Caption

Used for

- Metadata
- Dates
- Distances
- Status Information
- Copyright

Captions should remain readable.

---

# Overline

Used sparingly

Examples

- AI Recommendation
- Premium Feature
- Featured Journey

Avoid excessive overlines.

---

# Semantic Text Styles

Support

```
display

heading

title

subtitle

body

bodyStrong

caption

overline

label

button

link

code
```

Components should reference semantic styles,

never font sizes.

---

# Reading Rhythm

Maintain

- Comfortable Line Height
- Appropriate Paragraph Spacing
- Predictable Alignment
- Balanced White Space

Reading should feel effortless.

---

# Alignment

Default

Left Aligned

Center alignment

Only for

- Empty States
- Marketing
- Hero Sections

Avoid justified text.

---

# Text Length

Prefer

Short

↓

Clear

↓

Actionable

Long paragraphs should be avoided inside the application.

---

# Numbers

Numbers should align consistently.

Examples

- Budget
- Distance
- Time
- Ratings
- Journey Score

Financial values should always be easy to compare.

---

# AI Typography

AI Responses should

Highlight

- Recommendation
- Reason
- Confidence
- Actions

Users should scan AI responses quickly.

---

# Button Text

Buttons should use

Action verbs.

Examples

Good

```
Start Journey

Save Route

Add Stop

Compare Routes
```

Avoid

```
Click Here

Proceed

Submit

Execute
```

---

# Error Messages

Typography should reduce stress.

Good

```
We couldn't calculate your route.

Please try again in a few moments.
```

Bad

```
Route Error 503
```

---

# Forms

Labels should remain visible.

Helper text should explain.

Validation messages should be concise.

Typography should reduce mistakes.

---

# Tables

Support

- Numeric Alignment
- Consistent Headers
- Readable Row Height
- Responsive Layout

Large tables should remain easy to scan.

---

# Responsive Typography

Typography adapts across

- Mobile
- Tablet
- Desktop
- Large Displays

Hierarchy remains consistent.

---

# Accessibility

Support

- Dynamic Text Scaling
- WCAG AA Contrast
- Screen Readers
- Dyslexia Friendly Layout
- High Contrast Theme

Text should remain readable at every supported size.

---

# Localization

Typography supports

- Left-to-Right Languages
- Right-to-Left Languages
- Variable Character Width
- Multi-line Expansion

Translations should not break layouts.

---

# Typography Tokens

All typography uses tokens

```
font.family.primary

font.family.mono

font.display

font.heading.xl

font.heading.lg

font.heading.md

font.body.lg

font.body.md

font.body.sm

font.caption

font.label

font.button
```

Never define typography directly inside components.

Guide

- Load Quickly

- Support Variable Fonts

# Golden Rules

Never style text

without purpose.

Never create

unnecessary font variations.

Never sacrifice

readability for aesthetics.

Typography should make every journey easier to understand.
