# Purpose

This file is the single entry point into the documentation system.

---

# Documentation Principles

Documentation is considered the primary source of truth.

Before implementing new features:

1. Read the relevant documentation.
2. Confirm architectural consistency.
3. Update documentation if required.
4. Proceed with implementation.

Implementation should never contradict approved documentation.
