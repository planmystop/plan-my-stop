# Privacy

> Privacy is a core design principle of Plan My Stop. Users own their data. The platform only collects the minimum information necessary to provide a secure, intelligent, and personalized travel experience.

---

# Purpose

This document defines

- Data collection
- Data storage
- Data processing
- Data sharing
- User rights
- Data retention
- Privacy principles

Privacy should never be sacrificed for business growth.

---

# Privacy Principles

The platform follows these principles.

- Privacy by Design
- User First
- Transparency
- Data Minimization
- Explicit Consent
- Secure Processing
- User Control
- Regulatory Compliance

---

# Objectives

The privacy system should

- Protect user information
- Collect minimum data
- Give users control
- Secure personal information
- Support global privacy regulations
- Build user trust

# Data We Never Collect

The platform should never collect

- Passwords in plain text
- Credit Card Information
- Government Identification
- Private Conversations unrelated to journeys
- Personal Contacts
- SMS Messages

---

# User Consent

User consent is required before

- Accessing location
- Sending notifications
- Marketing emails
- Analytics cookies
- AI personalization

Users can withdraw consent at any time.

---

# Data Usage

Collected data may be used for

- Journey Planning
- AI Recommendations
- Personalization
- Account Management
- Platform Analytics
- Security
- Fraud Detection

User data is never sold.

---

# AI Data Usage

AI may receive

- Journey Details
- Preferences
- Anonymous Context

AI should never receive

- Passwords
- Payment Data
- Authentication Tokens
- Personal Secrets

Sensitive information must be removed before AI requests.

---

# Data Sharing

Data may be shared with

- Authentication Provider
- Payment Provider
- Email Provider
- AI Provider
- Maps Provider

Only the minimum required information should be shared.

---

# User Rights

Every user has the right to

- View Personal Data
- Download Personal Data
- Correct Personal Data
- Delete Personal Data
- Withdraw Consent
- Close Account

---

# Account Deletion

When a user deletes an account

↓

Personal Profile Removed

↓

Sessions Invalidated

↓

Personal Data Deleted

↓

Anonymous Analytics Preserved

↓

Deletion Complete

Legal obligations may require limited data retention.

---

# Data Retention

Recommended retention

| Data | Retention |
|--------|------------|
| User Profile | Until Account Deletion |
| Trips | Until Deleted by User |
| Reviews | Until Deleted |
| Logs | 90 Days |
| Analytics | Anonymous Only |
| Audit Logs | 1 Year |

# Best Practices

Always

- Collect minimum data
- Ask permission
- Explain why data is needed
- Allow deletion
- Protect user privacy

Never

- Sell user data
- Track users secretly
- Share unnecessary information
- Collect sensitive information without a valid purpose
